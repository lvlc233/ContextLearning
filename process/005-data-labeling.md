# 4. 数据标注
有别于机器学习领域的数据标注,上下文学习因为语言模型具备自然语言理解能力,因此在标注的过程中,可以通过多元的自然语言方式进行标注;

即可以简单的对 待数据标注对象标记 [好/坏]的标签
也可以详细到对每个输入项,乃至"参数"(例如上下文本体/修改内容/...)进行描述性补充说明;

一个案例是:
- 哪一句好；
- 哪一句不好；
- 为什么不好；

事实上有别于传统机器学习中若不进行数据标注就无法进行有效的学习过程;大语言模型是具备一定能力将某段上下文进行优化的能力的,但是这种优化能力依赖于原始上下文信息与模型本身偏好,常见现象就是:
初始轮次优化尚可,但是多轮优化后优化方向和内容开始呈现随机漂移与无效化优化;
多次独立优化总能找到优化点,尽管这些优化点在其他地方被视为良好内容;
总的来说呈现出一种"为了修改而修改"的惯性运动;
而这也正是真动态自上下文优化的困难所在;

因此在这一步,为上下文学习提供数据就是给定模型一个清晰的训练方向;(而有别于目标)




例如：

```text
“这一轮只检查一件事情”会让人有被控制感，不像引导。
```

这不是单纯的差评，而是一个很具体的偏好信号。

它可以转成规则：

```text
不要用控制式表达来限制用户。
如果需要聚焦，说明聚焦理由，而不是命令用户只能看一个点。
```
