# 训练

`上下文训练` 是指通过有方向性的指导，对模型上下文进行修改，并通过反馈信号逐步逼近 `训练目标` 的过程。

一个最基础的训练轮次如下：

```text
→ 加载待训练上下文
→ 输入数据集
→ 运行 agent
→ 评估偏离程度
→ 修改上下文
```

与 `机器学习` 不同的是，`上下文学习` 的训练周期要短得多，因此手动调整训练成为可能。实际上，通过某一个会话直接对 skill 提出迭代要求的过程，就可以视为一次手动训练过程。

在确定训练轮次的最小单位后，便可以不断重复这个过程，从而进行以批次为单位的 `上下文训练`。可以使用训练计划初始化训练环境，并手动或自动地执行训练任务，直到达到停止条件或进入测试评估阶段。

在训练过程中，至少需要注意以下问题：

- 足够的训练样本空间：防止过拟合，并加强训练底层而非表象。
- 平均有效的训练过程：避免偶发性幸运或失败破坏实际数据判断。
- 足够的训练次数：通过多次训练使信号逐步逼近目标，而不是期望一次训练就完成目标。
- 上下文局限性：对于同一个目标，不同起始条件（包括起始上下文）和不同推理模型会有不同表现；也就是说，相同上下文不一定完全适配不同模型。

## 如何评估偏离信号

在训练阶段，评估偏离信号的重点不是重新定义概念，而是判断一次运行结果是否提供了足够明确的训练依据。

通常可以按以下顺序进行：

1. 明确本轮训练目标。
2. 运行 agent，并保留原始输出。
3. 对照训练方向，观察输出中哪些行为更接近目标，哪些行为更偏离目标。
4. 标记偏离发生的位置、类型和严重程度。
5. 判断偏离是偶发输出，还是在多个样本或多次运行中稳定出现。
6. 将偏离信号整理为反馈信息，用于后续修改上下文。

例如：

```text
训练目标：
压缩上下文时，保留任务边界、失败条件和测试标准。

运行结果：
新上下文更短，但删除了失败条件。

偏离信号：
压缩行为达成，但关键约束丢失。

反馈信息：
当前压缩规则只强调减少重复表达，没有约束必要边界的保留。
```

在训练过程中，偏离信号不应该只记录成“好 / 不好”。它至少应尽量说明：

- 偏离了哪个训练目标；
- 偏离发生在哪个输出片段或行为上；
- 偏离是否与已有训练方向一致；
- 偏离是否稳定出现；
- 是否需要修改上下文，还是只需要继续观察。

只有当偏离信号能被转化为明确的反馈信息时，它才适合进入下一步上下文修改。

## 如何修改上下文

`上下文学习` 中涉及的训练单位是上下文本身。但这并不意味着一次训练过程必须接收一个完整的待训练上下文。也可以根据 `训练目标` 对上下文进行分层、切块或并发训练，以此调整单次训练过程中涉及的上下文信息。

同时，在训练过程中，应避免把 `训练目标`、`训练方向` 或单次标注原样塞进上下文本体。否则上下文可能只记住表层案例或评价语句，而没有学到可迁移的判断条件。

修改上下文可以是任意增删改操作。每次操作的幅度，取决于训练计划或训练过程中制定的信息。

通常来说，在运行完 agent 后，我们会根据模型输出的 `偏离信号` 整理出 `反馈信息`，再根据反馈信息调整上下文。

如果了解 `深度学习`，可以将这个过程类比为：

```text
正向传播 -> 模型输出 -> 计算梯度 -> 更新参数 ->... 
```

这个类比只是帮助理解。`上下文训练` 中被修改的是上下文本体，而不是模型参数；反馈信息也不是严格意义上的梯度。

## 可观测性

`上下文训练` 以上下文作为核心训练对象。这意味着训练目标、上下文本体、运行结果、反馈信号和修改记录等信息，大多可以被直接展开查看。

这与 `机器学习` 中大量信息隐藏在参数内部不同。在 `上下文训练` 中，可观测的内容通常更多，也更容易被人工检查、标注和干预。

下面列出一些常见的可观测内容。它们并不都需要在每轮训练中被人工检查，但了解哪些信息可以被观测，有助于判断训练过程中实际发生了什么，也有助于在训练偏离时及时干预。

常见可观测内容包括：

- **训练目标**：本轮训练试图逼近的行为。
- **上下文本体**：当前被训练、修改或比较的上下文内容。
- **输入样本**：本轮使用的训练样本或测试样本。
- **模型输出**：agent 在当前上下文下产生的原始输出。
- **偏离信号**：输出相对于训练目标或训练方向出现的偏差。
- **反馈信息**：由偏离信号整理出的修改依据。
- **修改记录**：本轮对上下文做了哪些增删改。
- **副作用**：某个修改是否破坏了其他目标或行为。


## 技巧

### 使用轻量化模型做近似评估

训练时可以优先使用同系列的轻量化模型进行近似评估。

同系列模型的训练数据、模型架构和行为模式通常更接近；使用轻量化模型可以降低训练成本，也更适合在早期训练阶段进行多轮试跑。

但这种缩放有边界。如果模型能力差距太大，或上下文能力差异过大，轻量化模型上的训练结果不一定能直接迁移到更强模型上。




## 后训练

后训练同样源于 `机器学习` 领域中的后训练过程。

在 `上下文学习` 中，后训练指的是：基于已经完成主要训练的 `上下文信息`，通过校准人类偏好、内容微调、安全声明、表达风格调整等方式继续校准上下文。

从素材来源上看，如果把一份已经写好的第三方上下文视为已有训练产物，那么后续围绕它进行的修改、适配和偏好注入，也可以视为后训练。

`后训练` 与普通 `上下文训练` 的操作很接近，但侧重点不同：后训练通常更适合处理偏好注入、细节调整、案例填装、内容表达风格等主观性更强的内容。

## checkpoint

该概念取自 `机器学习` 中同名的 checkpoint，指代在训练过程中存储中间训练结果，通常就是一个模型。而在 `上下文学习` 领域，checkpoint 往往可以包含更多信息。

checkpoint 至少应标记：

- 当前上下文版本号（用于标记）
- 上下文本体
- 训练目标偏离程度（类似于 loss）

由于 `上下文学习` 中 `上下文环境` 的特殊性，将训练用的 `上下文环境` 等信息存入 checkpoint 中也是一个可选方案。

checkpoint 的保留机制通常需要在 `训练准备阶段` 确认。可以通过训练轮次数量、是否深度靠近某一目标等信息，作为 checkpoint 保留的依据。

使用了 checkpoint 的另一个好处：

- 训练可回退：如果某一个版本的 checkpoint 表现不好，可以回退到指定版本继续使用，或调整训练策略，而无需一切从头开始。

### 使用 git 作为 checkpoint 存储工具

在进行“训练”过程中，可以选择 git 提交或 tag 记录每个 checkpoint。

如果使用多维度分区训练的方式，可以使用文件夹做内容隔离。

推荐以 checkpoint 作为单体，而不是以某次训练迭代作为提交单位。
