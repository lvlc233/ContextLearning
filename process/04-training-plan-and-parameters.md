# 设计训练计划与参数

在正式进入 `上下文训练`前，推荐设计好本次训练的计划与参数细节。当然，如果已有一个不错的训练计划，也可以直接复用。

在这个环节，我们将明确训练如何执行、使用哪些样本、如何记录结果、如何设置 checkpoint，以及每轮训练如何判断是否继续。

它回答的不是“上下文应该怎么改”，而是如何给`上下文训练`搭建一个好的环境，例如：

- 这轮训练测什么目标；
- 用哪些样本？
- 数据集的范围与测试用的占比
- 是否需要对照组和实验组
- 一轮跑几个样本、每个样本跑几次；
- 每轮观察什么？
- 怎么判断变好还是变坏？
- 输出结果用什么格式记录；
- 是否使用 subagent；
- 训练前怎么保留 baseline checkpoint；
- ...

这些内容可以类比为机器学习训练中的“超参数”，例如训练迭代方式、轮次、样本数量、采样次数、更改量、评估标准等。

但这个类比只是帮助理解。在上下文学习中，训练流程与参数不一定是程序字段，也可以是一套自然语言描述的实验协议。您也完全可以通过手动文档记录与执行的发生进行训练，但是如何没有描述清楚整个训练过程发生的内容，那么自动化就显得十分困难。尽管不是必要的，但是仍然建议，如果构建好了一套训练流程，可以尝试将其转化为与之对应的loop过程。

## 需要设计什么

一次上下文训练开始前，至少需要明确以下内容：

- **训练目标**：本轮上下文训练要逼近什么行为。
- **训练反馈信号**：判断接近行为的标准，用于校准训练后的内容是否真的靠近目标而不是随机探索。
- **样本范围**：使用哪些样本作为训练材料，哪些样本留作测试。
- **样本规模**：本轮先跑几个样本，是否需要多次采样观察稳定性。
- **更改范围**：只允许改表达，还是可以调整结构、删除规则或新增规则。
- **记录方式**：如何保存输入、输出、标注、版本和结论。
- **checkpoint**：正式修改前是否保留 baseline，方便后续对比和回退。


## 参数示例

这些参数可以根据训练目标调整：

| 参数 | 示例 |
| --- | --- |
| 样本数量 | 先选 2-3 个样本进行轻量测试 |
| 采样次数 | 每个样本生成 3-5 次，观察稳定趋势 |
| 对照组 | 使用旧上下文或 baseline 版本 |
| 实验组 | 使用修改后的上下文版本 |
| 记录内容 | 输入、旧输出、新输出、人类标注、副作用 |

## 边界

设计训练流程与参数不是训练本身。

它不直接负责修改上下文，而是为后续训练提供运行边界。真正的上下文修改发生在训练阶段；具体测试方法和评估细节则在测试与评估阶段展开。

一个好的训练流程不要求一开始就非常严密。早期可以先做临时实验，用来校准测试方式和评价维度。等流程稳定后，再逐步固化成可复用模板。
