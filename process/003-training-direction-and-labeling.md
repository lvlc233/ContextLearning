# 训练方向与数据标注

本步骤的目标是：通过指令式规则或数据标注，明确本轮 `上下文训练` 要使用的 `训练方向`。

对于 `训练方向` 而言，存在两种常见表现形式：

- **指令式训练方向**：直接写出正向信号、负向信号和判断规则。
- **标注式训练方向**：先对样本、输出或版本进行标注，然后直接投入训练。


## 指令式训练方向

指令式训练方向类似于奖励函数。

它不是通过样本标注间接形成方向，而是直接用自然语言描述什么是正向信号、什么是负向信号，以及修改上下文时应该优先保留或避免什么。

### 操作步骤

如果不知道从什么地方开始，可以按以下步骤进行：

- 明确本轮训练目标；
- 写出什么情况更接近目标；
- 写出什么情况偏离目标；
- 写出必要的优先级或冲突处理规则；
- 检查是否需要通过样本验证。

### 案例

训练目标：

```text
压缩上下文内容，同时保留任务边界、失败条件和测试标准。
```

指令式训练方向：

```text
压缩上下文时，删除重复表达是正向信号；
压缩上下文时，删除失败条件、任务边界或测试标准是负向信号；
如果简洁性和任务边界发生冲突，应优先保留任务边界。
```

指令式训练方向适合用于目标和偏好已经比较明确的场景。它的优点是清晰、直接，可以立刻参与上下文修改；风险是如果写得过于抽象或过早定死，可能会把尚未验证的偏好误写成规则。

## 标注式训练方向

标注式训练方向类似于从数据中提取奖励信号。

它不直接写出完整规则，而是通过对样本、输出、版本或案例进行标注，为 `上下文训练` 提供最小反馈信号。

### 操作步骤

如果不知道从什么地方开始，可以按以下步骤进行：

- 收集待标注样本；
- 标注输出好坏；
- 写明判断理由；
- 标记该判断是否可以泛化为训练方向。

### 案例

原始输出：

```text
新版本上下文更短，但删除了失败条件和测试标准。
```

标注：

```text
负向。压缩有效，但不能接受，因为它删除了失败条件和测试标准。
```

标注式训练方向适合用于需要通过样本观察逐步确认的场景。它的优点是有证据来源，能从实际输出中形成方向；风险是样本太少或标注不一致时，容易把单个案例误当成通用方向。



## 标注式到指令式转换

标注式方向可以转换为指令式方向。这种转换很常见，也适合在样本判断逐渐稳定后进行。

例如：

标注：

```text
负向。压缩有效，但不能接受，因为它删除了失败条件和测试标准。
```

这条标注可以形成指令式方向：

```text
压缩上下文时，删除重复表达是正向信号；
压缩上下文时，删除失败条件、任务边界或测试标准是负向信号。
```

## 边界

`训练方向` 不是 `训练目标` 本身，也不是普通案例堆积。

`数据标注` 不是为了把人类反馈逐字写进上下文，而是为了提炼能指导修改和评估的判断标准。

如果某条标注暂时无法泛化，应先保留为案例材料或偏好记录，而不是直接写成通用规则。
