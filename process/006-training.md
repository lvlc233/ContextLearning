# 训练

`上下文训练` 是指通过有方向性的指导,对模型上下文进行修改,并通过反馈信号逼近指定目标的过程。


一个最基础的训练过程如下
```text
→ 放入上下文中
→ 输入数据集
→ 跑测试
→ 评估偏离程度
→ 修改上下文
```
跑输出时要注意保留原始结果,通过可以使用会话信息进行保留;

将训练的每个过程都展开观测,有助于后续调整与分析;

在训练的过程中至少注意保持
- 足够的训练样本空间(防止过拟合 + 加强训练底层而非表象)
- 平均有效的训练过程(避免偶发性幸运与失败破坏影响实际数据)
- 足够的训练时间(多次训练使信号逼近而不是期望一次训练就可以完成目标)
- 上下文局限性,对于同一个目标对于不同的起始条件(包括起始上下文)和推理模型来说,会有不同的表现;也就是说相同的上下文对于不同的模型来说不完全适配

### 技巧
训练时优先使用同系列轻量化模型。

因为同系列模型在训练时候的使用数据与架构更加接近,使用轻量化模型可以降低训练所需成本

但这个缩放有边界。

如果模型能力差距太大，可能会存在其他模型不兼容现象。


在训练过程中,单次的训练可以分为单一目标的训练或组合训练,并不建议一次性对所有的目标进行训练,因为对所有目标进行同期训练可能会导致训练有效性需要长周期与多测试验证才能证明效果稳定优秀;

如果了解大模型moe架构的可参考moe架构进行上下文训练;




# checkpoint

该概念取自 `机器学习`中同名的checkpoint，指代在训练过程中中间存储训练结果，通常就是一个模型；而在 `上下文学习`领域，checkpoint往往可以有更多的信息。

checkpoint 至少标记：

- 当前上下文版本号(用于标记)
- 上下文本体
- 训练目标偏离程度(类似于loss)

由于 `上下文学习` 中 `上下文环境`的特殊性，将训练用的`上下文环境`等信息存入checkpoint中也是一个可选方案


## 使用git作为 checkpoint 存储工具

在进行 "训练" 过程中,可以选择git提交或tag记录每个checkpoint;

如果使用多维度分区训练的方式,可以使用文件夹做内容隔离;

推荐以checkpoint作为单体而非某次训练迭代作为提交单位
