# 训练

`上下文训练` 是指通过有方向性的指导，对模型上下文进行修改，并通过反馈信号逐步逼近指定目标的过程。

一个最基础的训练轮次如下：

```text
→ 放入上下文中
→ 输入数据集
→ 运行 agent
→ 评估偏离程度
→ 修改上下文
```

与 `机器学习` 不同的是，`上下文学习` 的训练周期要短得多，因此手动调整训练成为可能。实际上，通过某一个会话直接对 skill 提出迭代要求的过程，就可以视为一次手动训练过程。

在确定训练轮次的最小单位后，便可以不断重复这个过程，从而进行以批次为单位的 `上下文训练`。可以使用训练计划初始化训练环境，并手动或自动地执行测试任务，直到训练结束。

在训练过程中，至少需要注意以下问题：

- 足够的训练样本空间：防止过拟合，并加强训练底层而非表象。
- 平均有效的训练过程：避免偶发性幸运或失败破坏实际数据判断。
- 足够的训练次数：通过多次训练使信号逐步逼近目标，而不是期望一次训练就完成目标。
- 上下文局限性：对于同一个目标，不同起始条件（包括起始上下文）和不同推理模型会有不同表现；也就是说，相同上下文不一定完全适配不同模型。

## 如何评估偏离信号

在训练阶段，评估偏离信号的重点不是重新定义概念，而是判断一次运行结果是否提供了足够明确的训练依据。

通常可以按以下顺序进行：

1. 明确本轮训练目标。
2. 运行 agent，并保留原始输出。
3. 对照训练方向，观察输出中哪些行为更接近目标，哪些行为更偏离目标。
4. 标记偏离发生的位置、类型和严重程度。
5. 判断偏离是偶发输出，还是在多个样本或多次运行中稳定出现。
6. 将偏离信号整理为反馈信息，用于后续修改上下文。

例如：

```text
训练目标：
压缩上下文时，保留任务边界、失败条件和测试标准。

运行结果：
新上下文更短，但删除了失败条件。

偏离信号：
压缩行为达成，但关键约束丢失。

反馈信息：
当前压缩规则只强调减少重复表达，没有约束必要边界的保留。
```

在训练过程中，偏离信号不应该只记录成“好 / 不好”。它至少应尽量说明：

- 偏离了哪个训练目标；
- 偏离发生在哪个输出片段或行为上；
- 偏离是否与已有训练方向一致；
- 偏离是否稳定出现；
- 是否需要修改上下文，还是只需要继续观察。

只有当偏离信号能被转化为明确的反馈信息时，它才适合进入下一步上下文修改。

## 如何修改上下文

`上下文学习` 中涉及的训练单位是上下文本身。但这并不意味着一次训练过程必须接收一个完整的待训练上下文。也可以根据 `训练目标` 对上下文进行分层、切块或并发训练，以此调整单次训练过程中涉及的上下文信息。

同时，在训练过程中，应该避免在修改上下文时直接暴露训练目标与训练方向，以避免 TODO（机器学习领域应该有类似的表述）。

修改上下文可以是任意增删改操作。每次操作的幅度，取决于训练计划或训练过程中制定的信息。

通常来说，在运行完 agent 后，我们会根据模型对应输出的 `偏离程度` 生成 `反馈信息`，再让模型通过 `反馈信号` 调整上下文信息。

如果您了解 `深度学习`，可以将这个过程比喻为：

```text
正向传播 -> 模型输出 -> 计算梯度 -> 更新参数 ->... 
```

训练时优先使用同系列轻量化模型。

因为同系列模型在训练时使用的数据与架构更加接近，使用轻量化模型可以降低训练成本。

但这个缩放有边界。如果模型能力差距太大，可能会出现模型不兼容现象。

在训练过程中，单次训练可以分为单一目标训练或组合训练。并不建议一次性对所有目标进行训练，因为对所有目标进行同期训练，可能需要更长周期和更多测试，才能证明训练效果是否稳定且优秀。

## 后训练

### 后训练过程

这一步实际上与前期数据标注在上下文学习领域中的差距更小，本质上是对训练完或正在训练的上下文空间进行偏好性微调。它使用的技巧和方式，与前期标注和训练没有本质差异。

这个过程更像是一个人类专家或老师，在校准模型调整上下文空间的迭代方向，避免其偏离训练目标。

也可以在这个过程中，像进行 LoRA 训练一样进行细节微调。

在这个过程中，可以尝试直接从描述中令监督模型将原始输入转化为符合当前上下文空间的动作规则，并直接嵌入偏好。




# checkpoint

该概念取自 `机器学习` 中同名的 checkpoint，指代在训练过程中存储中间训练结果，通常就是一个模型。而在 `上下文学习` 领域，checkpoint 往往可以包含更多信息。

checkpoint 至少应标记：

- 当前上下文版本号（用于标记）
- 上下文本体
- 训练目标偏离程度（类似于 loss）

由于 `上下文学习` 中 `上下文环境` 的特殊性，将训练用的 `上下文环境` 等信息存入 checkpoint 中也是一个可选方案。

checkpoint 的保留机制通常需要在 `训练准备阶段` 确认。可以通过训练轮次数量、是否深度靠近某一目标等信息，作为 checkpoint 保留的依据。

使用了 checkpoint 的另一个好处：

- 训练可回退：如果某一个版本的 checkpoint 表现不好，可以回退到指定版本继续使用，或调整训练策略，而无需一切从头开始。

### 使用 git 作为 checkpoint 存储工具

在进行“训练”过程中，可以选择 git 提交或 tag 记录每个 checkpoint。

如果使用多维度分区训练的方式，可以使用文件夹做内容隔离。

推荐以 checkpoint 作为单体，而不是以某次训练迭代作为提交单位。
