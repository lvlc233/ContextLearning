# 训练

`上下文训练` 是指通过有方向性的指导,对模型上下文进行修改,并通过反馈信号逼近指定目标的过程。

一个最基础的训练轮次如下
```text
→ 放入上下文中
→ 输入数据集
→ 跑测试
→ 评估偏离程度
→ 修改上下文
```

与 `机器学习` 不同的是，`上下文学习` 的训练周期要短得多，因此手动调整训练成为可能（实际上通过某一个会话直接对skill提出迭代要求的过程就可以视为一个手动训练过程）

在确定了训练轮次的最小单位后，便可以不断的重复这个训练过程从而进行以批次为单位的`上下文训练`。使用训练计划以初始化训练环境手动/自动的执行测试任务直到训练结束。


而在训练的过程中至少注意考虑
- 足够的训练样本空间(防止过拟合 + 加强训练底层而非表象)
- 平均有效的训练过程(避免偶发性幸运与失败破坏影响实际数据)
- 足够的训练次数(多次训练使信号逼近而不是期望一次训练就可以完成目标)
- 上下文局限性,对于同一个目标对于不同的起始条件(包括起始上下文)和推理模型来说,会有不同的表现;也就是说相同的上下文对于不同的模型来说不完全适配

# checkpoint

该概念取自 `机器学习`中同名的checkpoint，指代在训练过程中中间存储训练结果，通常就是一个模型；而在 `上下文学习`领域，checkpoint往往可以有更多的信息。

checkpoint 至少标记：

- 当前上下文版本号(用于标记)
- 上下文本体
- 训练目标偏离程度(类似于loss)

由于 `上下文学习` 中 `上下文环境`的特殊性，将训练用的`上下文环境`等信息存入checkpoint中也是一个可选方案。

checkpoint的保留机制通常需要在`训练准备阶段`确认，您可以通过训练轮次数量，是否深度靠近某一目标...等信息作为checkpoint保留的依据。

使用了checkpoint的另一个好处：
- 训练可回退： 如果某一个版本的checkpoint表现不好，可以回退到指定的任意版本进行使用或调整训练策略，而无需一切从头开始。

## 使用git作为 checkpoint 存储工具

在进行 "训练" 过程中,可以选择git提交或tag记录每个checkpoint;

如果使用多维度分区训练的方式,可以使用文件夹做内容隔离;

推荐以checkpoint作为单体而非某次训练迭代作为提交单位
