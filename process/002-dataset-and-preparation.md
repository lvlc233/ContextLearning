# 数据集与准备

在 `上下文学习` 中，数据集的格式和内容没有严格限制。由于大部分 harness 都具备文档处理能力，最简单的数据集可以从一个 Markdown 文件开始。

数据集准备的重点不是格式复杂，而是要区分不同材料在训练过程中的用途。

一个最基础的数据单位是：

- Q：原始运行上下文；
- A：模型输出；
- T：人类标注（可选）。

## 数据类型

上下文学习中常见的数据类型包括：

- **原始数据集**：可用于人工数据分析阶段，包括但不限于 `训练目标` 的寻找、`训练方向` 的确定、`人类偏好` 的输出信号标记等。
- **训练数据集**：用于 `上下文训练` 过程。通常只取输入项的数据用于训练，并结合 `训练目标` 与 `训练方向` 对生成内容进行评估。运行过程中的数据集仍然可以作为 `原始数据集` 的一部分。
- **测试数据集**：用于评估训练后的上下文是否真的改善输出，是否产生了过拟合现象，在未见样本上是否稳定。
- **标注数据集**：带有人类判断、解释或修正意见的数据集合，用于提供更明确的训练信号。
- **偏好数据集**：标注数据集的一种，重点记录人类对不同输出、规则方向或上下文版本的偏好。
- **训练目标集**：记录本轮或多轮上下文训练要逼近的目标。它不是普通样本集，而是用于约束训练方向的目标集合。

这些数据集仅在功能作用上有实质差异，在内容形式上没有统一标准或严格差异。

## 案例

假设有一条来自真实运行的记录：

```text
Q：用户要求 Agent 压缩一份 skill，使它更短，但不能丢失任务边界、失败条件和测试标准。

A：Agent 删除了重复表达，也删除了失败条件和测试标准。

T：人类标注：输出确实更短，但丢失了必要约束，不能视为有效优化。
```

这条记录可以在不同阶段承担不同作用：

- 作为 **原始数据集**：用于观察上下文压缩时常见的失败模式。
- 作为 **标注数据集**：`T` 中的人类判断提供了明确训练信号。
- 作为 **偏好数据集**：它表达了“更短但不能丢失约束”的偏好。
- 作为 **训练目标集** 的来源：可以提炼出“压缩上下文时保留任务边界、失败条件和测试标准”这个目标。
- 作为 **训练数据集**：可以只取 `Q` 作为输入，让新上下文生成输出，再根据目标和标注判断是否改善。


## 一些技巧

### 使用自己的会话作为数据集

从实际的生产活动中筛选过滤出自己的数据集。

可以根据不同的对话内容，选择不同的数据单位：

- 以一次 run 记录作为数据单位，通常去除工具调用等细节；
- 以一组与指定任务相关的 run 记录作为数据单位；
- 以一个完整会话作为数据单位；
- 以一组输入、输出和人类评价组成一个样本。
