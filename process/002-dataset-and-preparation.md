# 1. 数据集与准备
最好的数据集就是从实际的生成活动中产出,它可以来自你本人的会话;
并尝试根据不同的对话内容,进行粗略的数据标注,常见的单位有:
- 以一次 run 记录作为数据单位(通常去除工具调用等细节)
- 以一组与指定任务相关的 run 记录为数据单位
- 以一个完整会话作为单位

关于数据集合的类型:

有常见的
- 训练数据集
- 测试数据集
- 标注数据集
    [
        通过人类标注少量输入输出对的信号,让模型拥有更好的标准用于进行上下文学习监督
    ]
    - 偏好数据集
        [
            同属于标注数据集的子类,常见于特定风格偏好,如前端美术细节实现风格的标注信息等;
        ]

与:
- 训练目标集
