# 测试与评估

测试与评估用于判断一次上下文修改是否真的改善了输出，而不是只在单个案例上看起来更好。

通常来说，为了验证训练内容是否具有泛化能力，在训练准备阶段就应该将数据集区分为 `训练集` 与 `测试集`。在完成训练后，可以加载指定 checkpoint 中的上下文信息，并在测试集中运行任务，获取实际输出后再进行校验。

测试与评估关注的不是“上下文是否被修改过”，而是：

```text
修改后的上下文是否更稳定地逼近 `训练目标`？
修改后的上下文是否只是在训练样本上变好？
修改后的上下文是否引入了新的副作用？
```

## 校验方式

上下文测试的校验方式可以有很多种，常见包括：

- **相似度验证**：比较输出与目标输出、参考答案或历史优质输出之间的接近程度。
- **目标评分验证**：根据训练目标或评估维度，对输出进行打分。
- **规则检查验证**：检查输出是否违反明确规则，例如是否泄露答案、是否删除必要边界。
- **对比验证**：比较旧上下文与新上下文在同一测试样本上的输出差异。
- **人类验证**：由人类判断输出是否更符合偏好、任务目标或使用场景。

这些校验方式可以组合使用，但每次评估应尽量说明本轮主要判断维度。

## 评估记录

一次测试与评估至少应记录：

- 使用的上下文版本或 checkpoint；
- 使用的测试样本；
- 使用的模型和工具环境；
- 输出是否更接近 `训练目标`；
- 是否只在训练样本上变好；
- 是否引入新的副作用。

## 边界

测试通过不等于上下文已经在所有场景中可靠。

测试只能说明：在当前测试集、当前模型、当前工具环境和当前评估标准下，上下文表现达到了预期范围。

如果测试样本过少、测试维度混杂、评估标准不清晰，测试结果很容易变成主观感觉，而不是有效证据。
