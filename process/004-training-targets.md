# 3. 制订 / 提炼训练目标

一次上下文训练要逼近的行为方向。

它应该来自真实问题和偏好信号，而不是一开始就写成抽象规则。

好的答案源于好的问题，好的训练源于好的目标。

尽管在机器学习领域，制定一个好的学习目标十分困难，但是在上下文学习方向，使用自然语言描述却相对简单许多。

常见的训练目标可以从人类偏好中提取：

- 行为习惯；
- 审美偏好；
- 表达风格；
- 信息密度；
- 规则优先级。

从经验中提炼出规律并总结成概念，再赋予解释意义，是常见的目标定义方式。

在定义训练目标时，建议考虑训练目标的独立性，即尽量避免多个训练方向互相干扰。只有这样，后续测试时才更容易判断是哪部分修改产生了效果。

建议建立一个专属的目标集合文档，用于明确训练目标。

## 建议

尽管上下文学习的训练目标门槛不算高，但是随便写一个目标只会导致训练出模糊混乱的上下文。

一个基本建议是：不要一开始就写规则，先确认问题到底是什么。

例如，问题不应直接写成：

```text
上下文效果不好。
```

更好的问题描述是：

```text
加载这份上下文后，Agent 经常忽略失败条件。
加载这份上下文后，Agent 会重复解释已经在 system prompt 中说明过的规则。
压缩上下文后，Agent 的输出更短，但任务边界丢失。
```

问题要尽量落到行为上。

坏的问题描述：

```text
反馈不好。
```

更好的问题描述：

```text
新版本上下文让 Agent 减少了冗余解释，但同时让必要的约束条件缺失。
```
