# 制订 / 提炼训练目标

`训练目标` 是一次上下文训练要逼近的行为方向。

在流程上，制订训练目标不是直接写规则，而是先把真实问题、偏好信号和失败现象整理成可观察、可测试的目标。

好的训练源于好的目标。如果训练目标模糊，后续的上下文修改、数据标注和测试评估都会变得难以解释。

## 来源

训练目标通常来自：

- 原始运行中的失败现象；
- 人类标注中的偏好信号；
- 测试输出中的稳定偏差；
- 上下文版本对比中暴露出的副作用；
- 对目标行为的显式描述。
- ...

## 提炼步骤

你可以有任意的方式去构建某一个特定任务或通用任务的训练目标集，其内容本身也并没有任何限制。

如果您不知道如何提取一个训练目标，那么可以尝试按以下顺序提炼训练目标：

1. 收集原始问题或人类反馈。
2. 判断它指向的是上下文内容问题，还是其他 Agent 工程问题。
3. 找出期望改变的具体行为。
4. 将抽象描述改写成可观察目标。
5. 检查目标是否能被测试。
6. 记录到训练目标集。

## 使用LLM辅助训练目标的提取

例如，人类反馈可能不是直接说“训练目标是什么”，而是说：

```text
这版上下文更短了，但是失败条件被删掉了。
```

可以让LLM将该原始反馈提炼成`训练目标`：

```text
压缩上下文时，保留任务边界、失败条件和测试标准。
```



## 表达方式

训练目标应该尽量落到 `行为表现` 上。

这里的行为不是指直接写出规则，而是指：加载这份上下文后，Agent 在输出、判断、选择或执行方式上会出现什么可观察变化。

不好的目标：

```text
提升上下文的效果。
让Agent 表现更好。
让这份 skill 更聪明。
```

更好的表达应该先写出偏差，再把偏差转成目标。

偏差描述：

```text
加载这份上下文后，Agent 经常忽略失败条件。
加载这份上下文后，Agent 会重复解释已经在 system prompt 中说明过的规则。
压缩上下文后，Agent 的输出更短，但任务边界丢失。
```

对应的训练目标：

```text
加载这份上下文后，Agent 在压缩内容时仍能保留失败条件、任务边界和测试标准。
加载这份上下文后，Agent 应减少对已说明规则的重复解释。
加载这份上下文后，Agent 应在保持输出精简的同时保留关键约束。
```

这些描述更适合进入训练流程，因为它们说明了：

- 问题发生在什么条件下；
- Agent 具体表现出了什么偏差；
- 后续可以用什么样的样本观察它；
- 标注和测试时应该判断什么。

训练目标不需要一开始就写成完美规则，但至少应该能让后续流程知道：期望什么行为、要看什么输出、判断什么偏差、朝哪个方向修改上下文。

## 独立性检查

制订训练目标时，需要检查目标是否过度混合。

例如：

```text
让模型生成的前端更好看。
```

这个目标可能同时包含视觉风格、布局密度、组件选择、信息层级、响应式表现等多个方向。直接用它训练，很难判断后续改动到底改善了什么。

更好的做法是先拆出本轮主要目标，例如：

- 提高组件视觉差异；
- 减少页面信息拥挤；
- 保持响应式布局不破坏；
- 增强品牌风格一致性。

一轮训练不一定只能有一个目标，但主目标应该明确。其他目标可以作为旁观测或副作用记录，而不能喧宾夺主。
