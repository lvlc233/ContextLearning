# 3. 制订/提炼 训练目标

一次上下文训练要逼近的行为方向。

它应该来自真实问题和偏好信号，而不是一开始就写成抽象规则。


好的答案源于好的问题,好的训练源于好的目标;

尽管在机器学习领域,制定一个好的学习目标十分困难,但是在上下文学习方向,使用自然语言描述却相对简单许多;

常见的训练目标就是从人类偏好中提取:
- 行为习惯
- 审美偏好
- 表达风格
...

从经验中提炼出规律并总结成概念,并赋予解释意义是常见的目标定义方式;

在定义训练目标的时候建议考虑训练目标的正交性,即互不干扰的训练方向,因为只有这样子才能够并发执行,并清晰的分清原因;

建立创建一个专属的目标集合文档,明确训练目标

### 建议
尽管上下文学习的训练目标门槛不算高,但是随随便便的一个目标也只会导致训练出一个模糊混乱的上下文;

一个建议是:

不要一开始就写规则。

先确认问题到底是什么。

例如这轮最早的问题不是：

```text
agent 不够苏格拉底。
```

而是：

```text
agent 在用户已经给出思路后，仍然机械提问；
没有先判断用户哪里对、哪里错；
问题有时候和用户当前注意力不对齐。
```

问题要尽量落到行为上。

坏的问题描述：

```text
反馈不好。
```

更好的问题描述：

```text
用户写了伪代码，但 agent 没指出具体哪一行的操作语义错了，而是转去问另一个问题。
```
