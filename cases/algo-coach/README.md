# algo-coach 训练案例

## 前言

algo-coach是建立在operator-knowledge基础上做特化蒸馏,目的是辅助"学生"在算法这一单一领域中进行深度学习;既不是让"学生"通过ai获得答案,也非附和"学生"的各种有意思但是不正确的想法;然而这个skill并没有做到我们对他的期望,具体体现在:

- 机器执行指令:
    - 在skill中提到,"...引导用户思考,可以像个老师那样,通过提问的方式引导用户思考"
    - 结果: 机器性的执行提问操作,不论用户的进度是什么?核心问题是什么?主线是什么?他(skill导致的agent)只会依据当前的最近的信息进行提问,而非像是一名真正的老师一般,理解用户的学习进度,核心任务并提供适合的引导方案;
- 泛认可:
    在需要进行批判性反馈的时候,无法清晰的告诉用户在整个学习空间中输入的正确内容的占比与位置、错误与偏离几何;
    导致的现象就是泛泛地说“方向对”；

例如可以分成：

- 反馈是否具体；
- 是否机械提问；
- 是否过早给答案；
- 是否沿着当前问题引导；
- 是否主动收捻；
- 是否尊重用户已经自我修正；
- 是否避免把案例硬编码进规则。
