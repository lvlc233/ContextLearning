## 训练目标

对 `训练目标` 的讨论从原有定义开始：

```text
训练目标 = 期望上下文加载后引发的行为集合。
```

用户进一步将其压到更底层：

```text
训练目标 = 训练中对模型行为的一组期望。
```

关键边界被确认：

- `训练目标` 是期望本身，不是对期望的描述。
- 文档中写出来的目标语句只是 `目标描述`，可能准确，也可能含混。
- “让模型更聪明”这种描述可能实际指向“不啰嗦”“指令遵循”“具有自主性”等不同期望。

因此，`显式目标 / 隐式目标` 被判断为不适合继续作为独立概念。依据是：显式或隐式描述的是目标是否被表达清楚，而不是目标本体是否存在。建议移除或降级为一句说明：

```text
训练目标不一定一开始就被清楚描述，它可能需要从偏好、失败现象、数据标注或版本对比中逐步提炼出来。
```

## 目标函数与目标诱发上下文

用户提出希望在上下文学习中存在一个类似机器学习 `目标函数` 的位置。经讨论确认：

```text
目标函数不是目标本身，而是目标的可优化表达。
```

在上下文学习中，直接把 `训练目标` 对标为目标函数并不准确，因为训练目标更像行为期望。用户随后提出 `目标上下文`，并进一步修正为更准确的 `目标诱发上下文/目标上下文`。

该概念的当前理解：

```text
目标诱发上下文指的是：假设存在一份理想的上下文信息，模型在指定上下文环境中加载它后，能够稳定产生符合训练目标的行为。它不是已经写好的上下文，也不是模型输出的信息本身，而是上下文训练试图逼近的理想上下文状态。
```

关键纠偏：

- 它不是某次输出结果。
- 它是被加载后能诱发目标行为的上下文本体。
- 类比机器学习时，训练的是函数/参数，不是函数输出；上下文学习训练的是上下文，不是某次输出。

示例：

```text
训练目标：
模型在回答代码评审请求时，优先指出具体风险，而不是先做泛泛总结。

目标诱发上下文：
一份被模型加载后，能稳定诱发这种评审行为的理想上下文信息。它可能包含评审优先级、输出顺序、风险判断标准和反例约束。训练时并不知道这份上下文最终应该怎么写，只知道需要不断逼近它，使模型加载后更稳定地先指出具体风险。
```

## 训练方向

对 `训练方向` 的初始梳理来自 `concepts/core/006-training-direction.md` 和 `process/003-training-direction-and-labeling.md`。原本它更接近“判断更接近或更偏离目标的标准”。

用户随后提出：`方向` 这个词可能更应对标机器学习中的 `梯度`，而不是 `loss` 或 `reward`。该判断被确认更准确。

当前关系被收束为：

```text
训练目标：期望什么行为。
目标诱发上下文：假设存在的、能诱发目标行为的理想上下文。
训练信号：一次输出/测试/标注提供的反馈依据，loss/reward 可类比为其中一种。
训练方向：根据训练信号推导出的上下文修改方向。
```

因此，原文中“训练方向类似奖励函数/奖励信号”的说法需要后续修正。更准确的方向是：

```text
loss / reward 更像训练信号；训练方向更像从这些信号中推出来的修改方向，接近梯度的位置，但不是数学梯度。
```

用户要求输出变短后，后续解释被压缩。关于梯度的当前短定义：

```text
loss 说：现在差多少。
梯度说：往哪边改会变好。
```

更抽象的表述：

```text
梯度是某个信号在当前状态下的局部变化方向。
```
