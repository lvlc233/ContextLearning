# 训练目标

`训练目标` 指当上下文信息被加载到 Agent 内部时，期望它引发的行为集合。

找到一个能够精确表达实际需求的 `训练目标`，是整个 `上下文学习` 过程中非常重要的一步。如果缺少 `训练目标`，或者对 `训练目标` 的描述不明确，整个系统就很难达到实际想要的效果。

缺少 `训练目标` 的 `上下文学习`，就像在没有靶心的情况下调参：上下文也许一直在被修改，但每一次修改都很难判断是接近目标，还是只是制造了新的偏移。这也是很多 `上下文学习` 失败的原因。

## 如何找到训练目标

很多失败的 `上下文训练`，问题通常不是规则太少，而是目标没有被看清。

例如：

```text
上下文让 Agent 的回复质量更好。
```

这不是一个好的训练目标，因为它太抽象，难以判断输出是否变好，也很难指导上下文应该如何修改。

更好的问题描述是：

```text
加载这份上下文后，Agent 应减少泛泛而谈的回复。
当任务已有明确输出标准时，Agent 应优先指出输入与标准之间的具体偏差。
当上下文规则之间发生冲突时，Agent 应优先遵循当前任务目标。
```

这些描述能落到行为上，也更容易测试。具体场景中的经验可以作为训练样本或证据，但不应直接等同于上下文学习领域的通用概念。

## 独立性

在指定训练目标时，应尽量确保 `训练目标` 具有一定的独立性。

例如：

- 反馈质量；
- 引导方向；
- 输出信息密度；
- 规则冲突处理；
- ...

基于独立性优先的训练目标有以下好处：

- 避免多个目标互相干扰；
- 方便观察评估；
- ...

某些目标实际上可能隐含多个子目标。如果直接执行训练，一轮训练中就会很难判断到底是哪部分修改起了作用。

例如：

- “我想要这段上下文可以让模型生成的前端更加好看。”

这个目标可能同时包含视觉风格、布局密度、组件选择、信息层级、响应式表现等多个方向。更好的做法是先拆出当前最需要训练的主要目标。
