# 训练目标

`训练目标` 指当上下文信息被加载到 Agent 内部时，期望它引发的行为集合。

训练目标可以被理解为一种关于期望行为的假设。它不一定在训练开始时就完全清楚，也不一定一开始就以明确规则的形式存在。

找到一个能够精确表达实际需求的 `训练目标`，是整个 `上下文学习` 过程中非常重要的一步。如果缺少 `训练目标`，或者对 `训练目标` 的描述不明确，整个系统就很难达到实际想要的效果。

缺少 `训练目标` 的 `上下文学习`，就像在没有靶心的情况下调参：上下文也许一直在被修改，但每一次修改都很难判断是接近目标，还是只是制造了新的偏移。这也是很多 `上下文学习` 失败的原因。

这里的“缺少训练目标”通常不是指完全没有任何意图，而是指训练目标尚未被显式化。它可能仍然隐含在人类偏好、样本表现、失败现象、数据标注或版本对比中。

## 如何找到训练目标

很多失败的 `上下文训练`，问题通常不是规则太少，而是目标没有被看清。

例如：

```text
上下文让 Agent 的回复质量更好。
```

这不是一个好的训练目标，因为它太抽象，难以判断输出是否变好，也很难指导上下文应该如何修改。

更好的做法是先写出偏差，再提炼训练目标。

偏差描述：

```text
加载这份上下文后，Agent 经常泛泛而谈，没有回应当前任务的具体条件。
当任务已有明确输出标准时，Agent 只给出笼统评价，没有指出输入与标准之间的具体偏差。
当上下文规则之间发生冲突时，Agent 会优先执行局部规则，而不是当前任务目标。
```

对应的训练目标：

```text
加载这份上下文后，Agent 应减少泛泛而谈的回复，优先给出和当前任务相关的具体判断。
当任务已有明确输出标准时，Agent 应指出输入与标准之间的具体偏差。
当上下文规则之间发生冲突时，Agent 应优先遵循当前任务目标。
```

这些目标能落到行为上，也更容易测试。具体场景中的经验可以作为训练样本或证据，但不应直接等同于上下文学习领域的通用概念。

## 显式目标与隐式目标

训练目标可以分为显式目标和隐式目标。

`显式目标` 指已经被写成可观察行为的目标。例如：

```text
加载这份上下文后，Agent 应指出输入与标准之间的具体偏差。
```

`隐式目标` 指还没有被明确写出，但已经存在于人类偏好、样本表现、数据标注、失败现象或版本对比中的目标。例如：

```text
这个回复虽然更短，但不如上一版会教。
这版看起来更稳定，但它太早给出了答案。
这个输出不差，但没有回应用户当前卡住的位置。
```

这些反馈或标注本身不是训练目标，但它们可能包含尚未显式化的训练目标。上下文训练的一部分工作，就是从这些训练信号中逐步提炼出更明确的目标。

因此，训练目标不一定总是训练的起点。它也可以是训练过程中被发现、修正和显式化的对象。

## 独立性

在指定训练目标时，应尽量确保 `训练目标` 具有一定的独立性。

例如：

- 反馈质量；
- 引导方向；
- 输出信息密度；
- 规则冲突处理；
- ...

基于独立性优先的训练目标有以下好处：

- 避免多个目标互相干扰；
- 方便观察评估；
- ...

某些目标实际上可能隐含多个子目标。如果直接执行训练，一轮训练中就会很难判断到底是哪部分修改起了作用。

例如：

- “我想要这段上下文可以让模型生成的前端更加好看。”

这个目标可能同时包含视觉风格、布局密度、组件选择、信息层级、响应式表现等多个方向。更好的做法是先拆出当前最需要训练的主要目标。

## 目标关系

训练目标不是简单的平铺列表。多个训练目标之间可能存在独立、组合或条件关系。

`独立目标` 指偏离信号可以相对单独观察和判断的目标。它适合用于训练早期，因为信号更清晰，更容易判断某次上下文修改是否真的改善了对应行为。

`组合目标` 指真实任务中需要同时成立的一组目标。组合目标更接近实际使用场景，但训练信号也更混杂，容易出现某个目标改善、另一个目标退化的情况。
