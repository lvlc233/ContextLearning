# 训练目标

`训练目标` 指当上下文信息被加载到 Agent 内部时，期望它引发的行为集合。

找到一个能够精确表达实际需求的 `训练目标`，是整个 `上下文学习` 过程中非常重要的一步。如果缺少 `训练目标`，或者对 `训练目标` 的描述不明确，整个系统就很难达到实际想要的效果。

缺少 `训练目标` 的 `上下文学习`，就像在没有靶心的情况下调参：上下文也许一直在被修改，但每一次修改都很难判断是接近目标，还是只是制造了新的偏移。这也是很多 `上下文学习` 失败的原因。

## 如何找到 训练目标
很多失败的`上下文训练`，问题通常不是规则太少，而是目标没有被看清。

例如：

```text
agent 不够苏格拉底。
```

这不是一个好的训练目标，因为它太抽象，难以判断输出是否变好。

更好的问题描述是：

```text
用户已经给出思路后，agent 仍然机械提问。
agent 没有先判断用户哪里对、哪里错。
```

这些描述能落到行为上，也更容易测试。

## 独立性
在指定训练目标时确保`训练目标`具有一定的独立性。

例如：

- 反馈质量；
- 引导方向；
- 输出信息密度:
- 连续性:
- ...

基于独立性优先的训练目标有以下好处
- 避免多个目标互相干扰
- 方便观察评估
- ...

某些目标实际上可能隐含了多个子目标，这时候如果执行训练，则在一轮训练的过程中，会产生如很难判断到底是哪部分的修改起作用的负面效果。

例如:

- "我想要这段上下文可以让模型生成的前端更加好看"
