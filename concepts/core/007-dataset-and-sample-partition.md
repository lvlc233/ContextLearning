# 数据集与样本划分

`数据集` 是在 `上下文学习` 中按用途组织起来的一组样本、记录或材料。

它的关键不在于文件格式，而在于用途和边界。

一个最基础的样本单位可以写成：

```text
Q：任务输入或原始运行片段
A：模型输出
T：人类标注（可选）
```

## 类型

常见的数据集类型包括：

- `原始数据集`：用于人工分析、提炼训练目标、形成训练方向和记录偏好信号。
- `训练数据集`：用于实际训练过程，通常以输入项为主。
- `测试数据集`：用于验证训练后的上下文是否真的改善输出，是否具有泛化能力。
- `标注数据集`：包含人类判断、解释或修正意见的记录集合。
- `偏好数据集`：标注数据集的一种，重点记录对不同输出、规则方向或上下文版本的偏好。
- `训练目标集`：记录本轮或多轮上下文训练要逼近的目标。

## 边界

同一条记录可以在不同阶段承担不同作用。

例如，一条来自真实运行的记录，既可以作为原始数据，也可以作为标注材料、偏好材料、训练目标来源或训练样本。

数据集之间的差异主要体现在用途上，而不是固定格式上。
