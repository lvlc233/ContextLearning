# 训练方向

`训练方向` 是上下文学习中非常重要的概念。

如果说 `训练目标` 回答的是：

```text
我想要什么？
```

那么 `训练方向` 回答的是：

```text
什么更接近我想要的？
什么是好的？
什么是坏的？
应该朝哪个方向修改上下文？
```

它类似于机器学习中的奖励信号或奖励函数，但在上下文学习中，训练方向不一定要写成程序函数，也可以由自然语言规则、人类标注、偏好样本或测试观察来表达。

## 与训练目标的区别

训练目标描述最终希望逼近的行为，训练方向描述如何判断当前修改是否正在靠近目标。

例如：

```text
训练目标：
压缩上下文时，保留任务边界、失败条件和测试标准。

训练方向：
如果新上下文更短，但删除了失败条件，则视为负向信号。
如果新上下文减少了重复表达，同时保留任务边界，则视为正向信号。
```

目标提供终点，方向提供每一步的判断标准。

如果只有训练目标，没有训练方向，模型可能会知道“要变好”，但不知道什么样的修改算变好。常见结果就是多轮优化后出现随机漂移，或者为了修改而修改。

## 来源

训练方向可以来自：

- 人类标注；
- 偏好比较；
- 失败案例；
- 版本对比；
- 测试结果；
- 人类明确写出的奖励规则。

这些材料不一定都要进入上下文本体。它们更常见的作用是帮助判断如何修改上下文，以及修改后是否更接近训练目标。




## 边界

训练方向不是训练目标本身。

训练方向也不应该被原样塞进上下文。人类标注、偏好判断和失败案例需要被提炼成可泛化的规则，否则容易导致上下文污染、过拟合或浅层模仿。
