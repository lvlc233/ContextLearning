# 数据标注

`数据标注` 是对样本、输出或案例做判断和说明的动作。


有别于传统机器学习中的数据标注，上下文学习中的标注可以使用自然语言完成。因为语言模型本身具备自然语言理解能力，所以标注不必只是一组离散标签。

它可以很简单：

```text
好 / 坏
更好 / 更差
有效 / 无效
```

也可以更具体：

```text
哪一句好；
哪一句不好；
为什么不好；
哪条规则导致了偏移；
哪个版本更接近训练目标；
是否出现副作用。
```

人类偏好也可以通过 `数据标注` 表达。它可以表现为对某个输出的直接评价，也可以表现为对两个版本的比较，或者对某种规则方向的接受与拒绝。

`数据标注` 是产生 `训练信号` 的一种常见方式。

标注本身不是训练目标，也不是上下文本体。它的作用是把判断、理由和偏好记录下来，供后续整理成训练信号、反馈信息或判断标准。

## 与训练方向的关系

`数据标注` 可以帮助形成 `训练方向`。

例如，一条标注如果明确指出：

```text
压缩有效，但不能接受，因为它删除了失败条件和测试标准。
```

这类判断就可以进一步整理成训练方向：

```text
压缩上下文时，删除失败条件、任务边界或测试标准是负向信号。
```

也就是说，标注不是直接等于规则，而是为规则提供依据。

## 边界

`数据标注` 不是把人类反馈逐字写进上下文。

`数据标注` 也不是 `训练目标` 本身。

如果一条标注还不能直接进入规则或上下文修改，就先把它保留为案例材料、偏好记录或训练信号记录，而不是强行写成通用规则。
