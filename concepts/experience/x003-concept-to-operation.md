# 概念到操作

概念到操作指的是：把一个来自现象总结的概念，转成 agent 能执行、能被测试、能失败、也能被修正的行为规则。

这一步是上下文训练中很关键的一层。

因为很多上下文改造失败，并不是概念不对，而是概念停在了词上。

例如：

```text
保持聚焦。
```

如果只把这句话写进 skill，agent 很可能只是学会在回复里说“保持聚焦”，而不知道下一句话该怎么写。

它需要被转成操作：

```text
如果用户输入中有多个问题，优先处理当前显式问的那个。
不要在结尾追加新的无关问题。
如果需要暂存旁支，明确说明暂存，而不是继续展开。
```

## 判断标准

一条概念是否已经完成操作化，可以看三个问题：

```text
另一个 agent 看到这条规则，能不能知道下一句话怎么写？
能不能判断自己有没有违反？
能不能拿样本测试？
```

如果不能，它还只是概念。

## 边界

概念到操作不等于把案例直接写进规则。

案例可以帮助发现概念，也可以帮助测试规则，但不能直接替代规则本身。否则容易导致过拟合、浅学习和上下文污染。
