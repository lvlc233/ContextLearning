# 偏好信号

在 agent应用领域中，尤其是在内容创作、内容风格分析等方面的todo

因此一个

偏好信号指的是：人类对一次输出、一次交互或一个规则方向给出的判断信息。

它不只是简单的“喜欢 / 不喜欢”，也可以是更细的描述：

- 哪一句不好；
- 为什么不好；
- 哪个动作让人有压力；
- 哪个问题偏离了当前注意力；
- 哪种表达更像引导，而不是控制。

## 与数据标注的关系

在上下文训练中，偏好信号通常来自数据标注。

例如：

```text
“这一轮只检查一件事情”会让人有被控制感，不像引导。
```

这不是单纯差评，而是一个可学习的偏好信号。

它可以转成规则：

```text
不要用控制式表达限制用户。
如果需要聚焦，说明为什么先看这个点。
```

## 与训练目标的关系

偏好信号不是训练目标本身。

偏好信号更像原始监督材料；训练目标是从多条偏好信号中提炼出来的方向。

例如多条反馈都指向：

```text
agent 没有先判断用户输入，而是继续提问。
```

那么训练目标才可以写成：

```text
用户给出思路后，agent 应先进行具体反馈，再决定是否追问。
```

## 边界

偏好信号不能被原样塞进上下文。

如果只是把每条用户反馈直接写进规则，模型可能会记住表层句子，而没有学到背后的判断条件。
