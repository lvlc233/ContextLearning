## 训练相关

| 名字 | 解释 | 案例 | 出自 |
|---|---|---|---|
| 训练目标/目标 | 在我们进行训练时，通常存在对 `模型行为`的一组期望，而这种期望就是`训练目标`，例如让”让模型更加的聪明“、“让模型更加的规范”...但是请注意，对`期望`的描述不等同于`期望`本身，例如对”让模型更加的聪明“的期望可能实际上表达的是“不啰嗦”、“指令遵循”、“具有自主性”....请注意`期望`与对`期望`的描述的区别 ｜ 目标：“让模型变得更加聪明”的期望；目标描述：“让模型变得更加聪明” | null |
| 目标诱发上下文/目标上下文 | 源于机器学习领域中 `目标函数`的概念，`目标上下文`指的是：假设存在一份理想的`上下文信息`，在指定的`上下文环境`中，能使模型加载完这段`上下文信息`后能够稳定的达到某个`目标`。它不是已经写好的上下文，而是上下文训练试图逼近的理想上下文状态。| 目标是“代码评审时先指出具体风险”；目标诱发上下文是一份被加载后能稳定诱发该评审行为的理想上下文信息，而不是某
  一次评审输出 |null|
| 训练方向 | 判断当前输出、上下文修改或版本对比是否更接近训练目标的局部标准。 | - | [core/006-训练方向 L?](core/006-training-direction.md) |

| 奖励信号 | 类比机器学习中的奖励信号；在上下文学习中可由自然语言规则、数据标注、偏好样本或样本观察表达。 | - | [core/006-训练方向](core/006-training-direction.md) |
| 训练反馈信号 | 训练信号在流程语境中的别名。 | - | [core/004-训练信号](core/004-训练信号.md) |
| 反馈信号 | 训练信号在流程语境中的简称。 | - | [core/004-训练信号](core/004-训练信号.md) |
| 偏离信号 | 模型输出、上下文修改或测试结果相对于训练目标、训练方向或人类标注出现偏差时，被观察到的证据。 | - | [core/004-训练信号](core/004-训练信号.md) |
| 正向信号 | 当前输出或修改更接近训练目标的信号。 | - | [core/004-训练信号](core/004-训练信号.md) |
| 负向信号 | 当前输出或修改更偏离训练目标的信号。 | - | [core/004-训练信号](core/004-训练信号.md) |
| 反馈信息 | 对训练信号的整理和表达，比原始信号更接近上下文修改，但仍不是最终规则。 | - | [core/004-训练信号](core/004-训练信号.md) |
| 偏好信号 | 偏好信号属于标记信号的一种，常见于初始化和后训练阶段，可转成训练方向、规则约束或测试标准。 | - | [理论迁移清单](theory-migration-list.md) |
| 标记信号 | 待补：通过标注、偏好或判断留下的信号类型；当前仅在偏好信号说明中出现。 | - | [理论迁移清单](theory-migration-list.md) |

## 数据与标注

| 名字 | 解释 | 案例 | 出自 |
|---|---|---|---|
| 数据标注 | 对样本、输出或案例做判断和说明的动作；在上下文学习中可以使用自然语言完成。 | 好 / 坏；更好 / 更差；有效 / 无效 | [core/003-数据标注](core/003-数据标注.md) |
| 人类偏好 | 人类对输出、版本、规则方向或行为表现的接受、拒绝、比较和评价，可通过数据标注表达。 | - | [core/003-数据标注](core/003-数据标注.md) |
| 数据集 | 在上下文学习中按用途组织起来的一组样本、记录或材料。 | - | [core/007-数据集与样本划分](core/007-dataset-and-sample-partition.md) |
| 样本单位 | 一个基础样本可以包含任务输入或原始运行片段、模型输出和可选的人类标注。 | Q / A / T | [core/007-数据集与样本划分](core/007-dataset-and-sample-partition.md) |
| 原始数据集 | 用于人工分析、提炼训练目标、形成训练方向和记录偏好信号的数据集。 | - | [core/007-数据集与样本划分](core/007-dataset-and-sample-partition.md) |
| 训练数据集 | 用于实际训练过程的数据集，通常以输入项为主。 | - | [core/007-数据集与样本划分](core/007-dataset-and-sample-partition.md) |
| 测试数据集 | 用于验证训练后的上下文是否改善输出、是否具有泛化能力的数据集。 | - | [core/007-数据集与样本划分](core/007-dataset-and-sample-partition.md) |
| 标注数据集 | 包含人类判断、解释或修正意见的记录集合。 | - | [core/007-数据集与样本划分](core/007-dataset-and-sample-partition.md) |
| 偏好数据集 | 标注数据集的一种，重点记录对不同输出、规则方向或上下文版本的偏好。 | - | [core/007-数据集与样本划分](core/007-dataset-and-sample-partition.md) |
| 训练目标集 | 记录本轮或多轮上下文训练要逼近的目标。 | - | [core/007-数据集与样本划分](core/007-dataset-and-sample-partition.md) |
| 偏好形成集 | 已参与规则形成的样本集合，不能再强证明泛化能力。 | - | [理论迁移清单](theory-migration-list.md) |
| 未见测试集 | 未参与规则形成的样本集合，用于检查上下文修改是否泛化。 | - | [理论迁移清单](theory-migration-list.md) |

## 经验概念

| 名字 | 解释 | 案例 | 出自 |
|---|---|---|---|
| 训练信号经验 | 训练信号在实际训练中的经验性用法，不重复基础定义。 | - | [experience/004-训练信号经验](experience/004-training-signals.md) |
| 案例优先 | 训练过程优先把案例直接写入上下文，而不是先提炼案例背后的判断条件；这是一种反例信号。 | - | [experience/004-训练信号经验](experience/004-training-signals.md) |
| 案例风险 | 案例可用于暴露失败和形成偏好，但直接写入上下文容易导致过拟合、浅学习和上下文污染。 | - | [理论迁移清单](theory-migration-list.md) |
| 过拟合 | 上下文过度贴合已有案例或样本表层，导致泛化能力下降。 | - | [experience/004-训练信号经验](experience/004-training-signals.md) |
| 浅学习 | 模型学到案例或规则的表层形状，而不是背后的判断条件。 | - | [experience/004-训练信号经验](experience/004-training-signals.md) |
| 上下文污染 | 把未经提炼的案例、反馈或局部偏好直接写入上下文，导致上下文变脏、变偏或互相冲突。 | - | [experience/004-训练信号经验](experience/004-training-signals.md) |
| 直接修正 | 把输入端反馈直接嵌入上下文的做法；较好的方式是从反馈中学习方向，再转成可泛化的上下文信息。 | - | [experience/004-训练信号经验](experience/004-training-signals.md) |
| 动作信号 | 用可观察动作来表达或拟合某一个概念或需求。 | 删除重复表达；合并语义相近的规则 | [experience/004-训练信号经验](experience/004-training-signals.md) |
| 操作性 | 一条规则能让另一个 agent 知道下一句话怎么写、能判断是否违反、能拿样本测试。 | - | [experience/004-训练信号经验](experience/004-training-signals.md) |
| 概念到操作 | 把现象总结出的概念，转成 agent 能执行、能被测试、能失败、也能被修正的行为规则。 | 保持聚焦 -> 优先处理当前显式问题 | [experience/x003-概念到操作](experience/x003-concept-to-operation.md) |
| 反馈质量 | agent 能否贴着用户的真实输入，具体说明用户当前理解的状态。 | 指出哪一句对、哪一行伪代码有问题 | [experience/006-反馈质量](experience/006-feedback-quality.md) |
| 尺子式反馈 | 一种具体反馈方式：先指向用户输入中的具体位置，再说明为什么影响正确性。 | 先指出具体代码行表达了什么 | [experience/006-反馈质量](experience/006-feedback-quality.md) |
| 收束 | 反馈末尾的总结动作；总结也应贴具体对象，而不是泛化肯定。 | - | [experience/006-反馈质量](experience/006-feedback-quality.md) |
